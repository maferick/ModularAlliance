# Admin Member Audit Workflow

This guide covers how admins/HR operate the Member Audit dashboard.

## 1. Configure Scope Policy

1. Go to **Admin → Corp Tools → Scope Policy**.
2. Define required scopes for audits.
3. Optionally add optional scopes for user opt-in.
4. Save and activate the policy.
5. Add overrides for special groups or individual pilots.

## 2. Monitor Compliance

1. Open **Admin → Member Audit**.
2. Use filters (name, corp/alliance ID, group, status).
3. Review summary cards for compliance and token issues.

## 3. Trigger Audits

1. Select members.
2. Click **Trigger audit for selected**.
3. Review results in the expanded detail panels.

## 4. Request Re-Auth

1. Select members with missing scopes or token issues.
2. Click **Generate re-auth links**.
3. Share the links with affected users.

## 5. Export Results

Use **Export CSV** for offline review or reporting.
